Why do the default parameters make biological sense?

The simulator ships with a default parameter set chosen to reflect realistic bacterial gene-circuit behavior rather than arbitrary numbers, and each gate is built from a Hill-function ODE model. Jump to a section below.

Model equations Parameter defaults Logic gates

Model equations

The simulator is built on three core equations: one for how each logic gate produces protein, one general framework for how circuit burden slows bacterial growth, and the simplified version of that framework that is actually implemented in this simulator.

Equation 1 — Protein production from gene logic gate
dP/dt  =  β · f(inputs)  −  γP

Models protein concentration over time: rate of protein change = protein production − protein loss.

Equation 2 — Growth rate affected by burden
μ  =  μmax / (1 + B/k)

Models the bacteria's actual rate of growth by weighing three things together: the cell's max growth rate, the burden the circuit places on the cell, and how sensitive growth is to that burden.

Equation 3 — Direct protein model used in this simulator
μ  =  μmax · max(0, 1 − P/Pcap)

Models how protein expression decreases growth directly, by substituting the protein concentration P in for the abstract "burden" B from Equation 2.

Circuit & protein parameters

beta = 5.0maximum protein production rate

A moderate-to-strong promoter — strong enough to produce a visible, meaningful biological signal without overwhelming the cell with excessive output.

gamma_deg = 0.050basal protein degradation rate

Corresponds to a protein half-life of about 14 minutes (ln(2) / 0.05), a realistic value for a degrading protein model.

K = 1Hill sensitivity threshold

The tipping point where the gate flips from "off" to "on"; conventionally set to 1 in logic-gate models.

n = 2Hill cooperativity coefficient

Controls how sharp the on/off transition is. Transcription factors typically bind DNA as dimers, making a value of 2 experimentally realistic.

mu_max = 0.020unburdened max growth rate

Close to the textbook E. coli doubling time of 35 minutes (ln(2) / 0.02) in rich media. This anchors the growth rate before accounting for circuit burden.

P_cap = 20resource ceiling

Four times the max production rate (beta = 5.0), so a single gate is nowhere near lethal, but a stack of gates could push past the ceiling and slow metabolism.

Input parameters

inputLevel = 3A, B, R input concentration

Inputs default to a few multiples of K (over the sensitivity threshold) to ensure gates actually saturate into an on/off state rather than sitting in a middle zone.

Initial conditions

N0 = 1starting population

Some amount of bacteria is needed to start the simulation, so the default is one unit of starting bacterial population.

P0 = 0starting protein concentration

At time = 0 minutes, no protein has yet been produced by the starting bacterial population.

This is a simplified Hill-function model, not a full biochemical simulation — steady state here is read off a long-run simulation rather than solved analytically.

NodeWired inputsFinal PShare of burden
Model: every gate node still follows dP/dt = f_gate(inputs) · beta − gamma·P, but inputs may now be another node's live protein trajectory (fed through the same Hill activation/repression function) instead of a fixed step signal. gamma = mu(t) + gamma_deg and mu(t) = mu_max·max(0, 1 − ΣP / P_cap) are shared across every node, and dN/dt = mu(t)·N drives one colony. The whole system — every node's protein state plus the shared population — is integrated together with fixed-step RK4 (dt = 0.5 min), which is what allows feedback loops (a node wired back into an earlier node) to work correctly, not just simple chains.

Variable glossary

Circuit parameters

β beta
Maximum production rate of the reporter protein when the gate is fully switched on. a.u./min
γ_deg gammaDeg
Basal protein degradation rate — decay that happens independent of cell growth/dilution. 1/min
K K
Hill threshold — the input concentration at which the regulatory response is half-maximal. a.u.
n n
Hill coefficient — controls how steep/switch-like the activation or repression response is as input crosses K. unitless
μ_max muMax
Maximum bacterial growth rate, reached when the circuit imposes no protein burden (P = 0). 1/min
P_cap Pcap
Resource ceiling — the protein level at which cell growth is driven to zero because the circuit has consumed the cell's resources. a.u.

Regulatory inputs

A, B, R inputLevel
Shared concentration used for whichever input signals are active (activators A/B, or repressor R for the NOT gate) once switched on. a.u.
R_on rOn
Simulated time at which the repressor input R turns on for the NOT gate (a step function). min
A_on aOn
Simulated time at which input A switches on for the two-input gates (AND, OR, NAND, NOR, XOR). min
B_on bOn
Simulated time at which input B switches on for the two-input gates. min

Initial conditions

N₀ N0
Starting bacterial population size at t = 0, before any simulation runs. a.u.
P₀ P0
Starting reporter protein level at t = 0, before any simulation runs. a.u.

Simulated / derived quantities

t
Simulated time since the run began, integrated from 0 to 300 in fixed 0.5-min steps (RK4). min
P(t)
Reporter protein output over time for the selected gate — this is the fluorescence signal shown in the colony readout and output chart. a.u.
N(t)
Bacterial population size over time, which grows more slowly as circuit burden (P) increases. a.u.
μ(t)
Instantaneous growth rate at time t, computed as μ_max · max(0, 1 − P/P_cap) — it falls as protein burden rises toward P_cap. 1/min
γ
Total effective removal rate of protein: γ = μ(t) + γ_deg, combining dilution from growth and basal degradation. 1/min
